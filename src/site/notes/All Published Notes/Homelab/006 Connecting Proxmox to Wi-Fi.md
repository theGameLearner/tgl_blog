Due to the issue I faced in changing my router, I want to be able to allow my Proxmox to connect to my WiFi router using wireless as well, so I can use that in a pinch if i ever re-locate having same Router but unable to connect to it using a LAN. 

Connecting Proxmox over WiFi is unadvised at-least by Proxmox[^1], they claim there are several disadvantages, at least the following disadvantages apply:
- Wi-Fi adapters can only be used as Linux bridge interface through workarounds, as most Access Points (APs) will reject frames that have a source address that didn’t authenticate with the AP.
- compared to wired Ethernet connections you will experience more latency spikes, reduced bandwidth and depending on distance and barriers between host and the AP even spotty connections

> [!Warning]
> We are only connecting WiFi to the proxmox machine for troubleshooting, this does not allow my VMs to work directly with the WiFi connection, for allowing these, I need other work arounds.

While my router and Proxmox is connected using LAN, I need to do the setup.

###### Install wpasupplicant
Then I want to install `wpasupplicant` to use for my Proxmox machine. This is the software suite for handling Wi-Fi connections on Linux.
```bash
root@pve:~# apt install wpasupplicant
```

As I had updated the masks, the file `cat /etc/resolv.conf` had wrong mask leading to issues in reaching 'deb.debian.org', make sure to fix the issue before proceeding.
###### Configure wpa_supplicant
This step configures the connection to your specific Wi-Fi network. Replace `[YOUR_SSID]` and `[YOUR_PASSWORD]` with your actual network name and password:
```bash
wpa_passphrase [YOUR_SSID] [YOUR_PASSWORD] > /etc/wpa_supplicant/wpa_supplicant.conf
```

If you want to directly type in the file, you need to type using the right formatting which is auto generated by `wpa_passphrase`.
For example, my SSID (wifi connection name) is 'Airtel_vidh_2470' and the WiFi password is 'Air@13359':
```bash
root@pve:~# # entered wrong text in the file
root@pve:~# cat /etc/wpa_supplicant/wpa_supplicant.conf
wpa_passphrase Airtel_vidh_2470 Air@13359
root@pve:~# # delete and re-create the file
root@pve:~# rm /etc/wpa_supplicant/wpa_supplicant.conf
root@pve:~# cat /etc/wpa_supplicant/wpa_supplicant.conf
cat: /etc/wpa_supplicant/wpa_supplicant.conf: No such file or directory
root@pve:~# wpa_passphrase "Airtel_vidh_2470" "Air@13359" > /etc/wpa_supplicant/wpa_supplicant.conf
root@pve:~# # print correct file
root@pve:~# cat /etc/wpa_supplicant/wpa_supplicant.conf
network={
        ssid="Airtel_vidh_2470"
        #psk="Air@13359"
        psk=39599dd2a4ffc21cd939634bd4186f4db6b72875fd0559aae793f893f903a0e3
}
root@pve:~# 
```
You need not use double quotes (`"`) if your SSID and passwords are a single word.

###### Connect using WiFi
In my case, the `ip a` gave me the WiFi interface name as `wlp41s0` so i will use this to configure and connect to my WiFi.
Steps:
- `ip link set wlp41s0 up`: up the link 
- `wpa_supplicant -B -i wlp41s0 -c /etc/wpa_supplicant/wpa_supplicant.conf`: initialize the connection
- `dhclient wlp41s0`: request an IP address via DHCP
- `ip a show wlp41s0`: see the local IP address for wlp41s0 client
- `ping -c 2 8.8.8.8`: test by pinging

```bash

root@pve:~# wpa_supplicant -B -i wlp41s0 -c /etc/wpa_supplicant/wpa_supplicant.conf
```

Now, our Proxmox host is connected to WiFi and can be used if the Ethernet is ever disconnected.

###### Persist the connection on re-boot
Next, we want to ensure that if we re-boot the machine, we can auto connect to the same WiFi.

We want to create a configuration file which will tell the machine which interface is used in which fashion(`/etc/network/interfaces`):
```bash
auto wlp41s0
iface wlp41s0 inet dhcp
    wpa-conf /etc/wpa_supplicant/wpa_supplicant.conf
```

For this, we create a 'systemd' service for `wpa_supplicant` which will start and use `/etc/network/interfaces` to get DHCP on `wlp41s0` automatically.
```bash
systemctl enable wpa_supplicant@wlp41s0.service
systemctl start wpa_supplicant@wlp41s0.service
```

Practical use code:
```bash
root@pve:~# # old value
root@pve:~# cat /etc/network/interfaces
auto lo
iface lo inet loopback

iface nic0 inet manual

auto vmbr0
iface vmbr0 inet static
        address 192.168.1.101/24
        gateway 192.168.1.1
        bridge-ports nic0
        bridge-stp off
        bridge-fd 0

iface nic1 inet manual


source /etc/network/interfaces.d/*
root@pve:~# fresh /etc/network/interfaces
root@pve:~# # new value

root@pve:~# systemctl enable wpa_supplicant@wlp41s0.service
Created symlink '/etc/systemd/system/multi-user.target.wants/wpa_supplicant@wlp41s0.service' → '/usr/lib/systemd/system/wpa_supplicant@.service'.
root@pve:~# systemctl start wpa_supplicant@wlp41s0.service
root@pve:~# 
```


If needed, use `systemctl restart networking` to use updated settings.

###### Accessing VMs and LXCs through WiFi
As I am not trying to have my homelab connected over WiFi, I am not covering how to set up to allow my VMs and LXCs to connect to the setup over Wifi, but making only a note here for future use if needed:
My options:
- *Masquerading (NAT)*: Configure your Proxmox host to act as a router, hiding your VMs behind its own IP address. This is more complex but a common solution.
- **4-Address Mode (WDS)**: This is the ideal solution if your **router and Wi-Fi adapter both support WDS**, as it allows for a transparent bridge. However, support is not guaranteed and getting it to work can be difficult
